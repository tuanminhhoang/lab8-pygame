Main Game Loop Architecture
The core loop runs at 60 FPS. Each frame: events → update world → draw → repeat.
Function Call Hierarchy
run() orchestrates: handle_events() → update_world() → draw_world() → text overlay.
Per-Frame Sequence
Clock measures dt, events processed, world updated, frame rendered and displayed.
Data Flow: moving_vector
Velocity created → modified by steering and borders → used for position integration.
Steering Behavior: Flee
Detect larger threats → compute flee vector → clamp speed → return safe velocity.

✅ Good Patterns

Game Loop Modularity
Basics
Separate handle_events(), update_world(), draw_world(). Each function does one job.
Engineering Insight
This separation enables testability and independent refactoring of rendering vs. physics.
Architecture Insight
Professional game engines (Unity, Unreal, Godot) follow this event-update-render pattern as their core loop.
Delta-Time Physics
Basics
position += velocity * dt makes movement frame-rate independent.
Engineering Insight
Without delta-time, higher FPS → faster gameplay. With it, physics is predictable at any frame rate.
Architecture Insight
Essential for networked games (determinism) and debuggers (frame-by-frame stepping).
Vector Math Abstraction
Basics
Use Vector2 instead of manual x/y math. Built-in: normalize(), distance_squared_to().
Engineering Insight
Reduces bugs; code reads like math instead of scalar soup.
Architecture Insight
Vector2 is implemented in C. Math operations are 10–100× faster than hand-rolled Python.

⚠️ Potential Issues

Border Recovery Has Directional Bias
Basics
free_from_border() adds or subtracts fixed velocity near edges, which can bias motion over time.
Engineering Insight
The edge correction is not symmetric in effect and can push squares into repeated right/down recovery behavior.
Architecture Insight
Border recovery and steering are coupled through direct velocity mutation, so the motion model is harder to tune independently.
Missing Return Type on Threat Lookup
Basics
distance_to_others() returns a square, but its signature does not state that explicitly.
Engineering Insight
The type hints are mostly strong, but this one missing return annotation still weakens static checking and autocomplete.
Architecture Insight
Small annotation gaps matter more in helper-heavy code because they make the object flow less explicit.

🚀 Performance Wins

Squared Distance Optimization

distance_squared_to() avoids sqrt when ranking. Saves work when picking the nearest larger square each frame.

Vector2 Native Implementation

Pygame's Vector2 is C-based. Math is 10–100× faster than hand-rolled Python.

🐢 Performance Risks

Quadratic Threat Detection

Every square iterates all squares: O(n²). At 30 squares, manageable. At larger counts, it becomes the bottleneck.

List Rebuilding Each Frame

larger_squares() and distance_to_others() still rebuild temporary lists every frame.
